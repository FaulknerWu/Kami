# ADR-0004: 读者端前台 Mock 缺口与下一阶段补齐计划

## 状态

Proposed

## 上下文

当前已将读者端前台接入现有公开接口：

- `GET /api/posts`
- `GET /api/posts/{slug}`
- `GET /api/categories`
- `GET /api/tags`

但原型中仍有一部分信息和交互暂无后端支撑。如果直接散落在页面里，后续替换成本会持续上升，因此需要集中记录并显式标记。

## 决策

第一阶段先允许以下内容保留为 Mock，并统一集中在前端可检索位置，同时在 DOM 或代码里添加 `ADR-0004` 标记：

- 站点作者资料：头像、封面、昵称、简介
- 关于页正文与联系方式
- 顶部搜索按钮
- 文章详情页分享按钮
- 阅读时长展示

其中：

- 作者资料与关于页内容统一集中在 `frontend/lib/mocks/site-profile.ts`
- 无真实交互能力的按钮统一保留 `data-kami-mock-feature` 标记
- 阅读时长当前按摘要长度估算，避免列表页与详情页不一致

## 后续工作

下一阶段优先补齐以下能力：

1. 站点资料公开接口
   建议提供站点名称、作者简介、头像、封面、联系方式等字段
2. 关于页内容来源
   建议确定为独立接口或并入站点资料接口
3. 搜索能力
   建议新增公开搜索接口，并明确分页、关键词高亮与排序规则
4. 分享能力
   需要明确是纯前端 Web Share / 复制链接，还是后端短链方案
5. 阅读时长
   需要明确是在后端预计算返回，还是继续由前端基于正文估算

## 结果

当前前端可以上线读者端主体浏览能力，但仍带有可追踪、可替换的临时 Mock 区域，不视为最终完成态。
